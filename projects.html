COMP 1502 Assignment 3

Technical

The technical objective was uplifting me and my team's code from assignment 2 and creating a GUI as the interface for the assignment instead of using the terminal. After splitting up the work and delegating, I was tasked with doing the third tab of the assignment, which was removing a toy through the serial number.

Some of the technical challenges me and my group experienced was the trying to work with JavaFX and Scenebuilder, getting it to work in the first place. However, when we finished setting up the scenes, everything else came naturally into place for me at least.

Design

Reviewing the rubic of the assignment, there wasn't any marks given towards how nice the interface looked. Therefore, I followed the suggested format, which I think turned out pretty well.

Challenges

(Refer to the code snapshot) The biggest issue that stumped me for a good bit was whenever I ran the remove serial number, any subsequent requests yeilded a blank arraylist. As I coded it to search through the arraylist object then remove that object with the matching specifications, in this case, the serial number.

However, when it removed it and ran the updateListView(), it completely wiped the object arraylist, turning it into an empty one instead. When I was coding it, I was testing with true/false iterations, as I was soley convinced that the issue was in the removeSerialNumberTab3() function. And not the real issue, of it being in updateListView().

When I took a break and came back, I decided to look over the code as a whole and see the logical process. That's when I found the bug in the updateListView() where it wipes the object arraylist. And so, that's how I solved the biggest hurdle for me in COMP 1502 assignment 3.

What Could be Improved

What could be improved is whenever I have a bug arise in the program, I take the time to follow the function logically, including if it calls another function. The result is that I avoid chasing red herrings and getting frustrated over a working function, when the real bug is in another one instead.

COMP 1502 Assignment 2

Technical

The technical requirements of assignment 2 was reading a text file with toy specifications, putting them into 4 unique objects, then adding them all into one object arraylist to be purchased or removed. As the assignment specifictions said that we were writing code for a toy store.

My role in this assignment was the loading content from the text file and saving from the object arraylist back into a textfile.

I loaded the text file, then I delimited it by using the split function, then I called the function ToyConstructor() to organize the toys from the textfile into an object arraylist. I created another function call to avoid bloating the current function, and to make it easier to read whenever an error occurs.

Then once the ToyConstructor() was called, it created the specifc toy object and then loaded it into the object array called "rows".

Design

I'm happy with the overall design of the functions I created. As I tried my best to make sure the code is readable but also is easy to come back to in the future.

Challenges

The biggest challenge on my part of assignment 2 was saving to a text file. Intially, I thought I could simply pull specifc variables from the object array then load them into the text file. It didn't work which caused me some frustration, however, what I failed to realize is that in an object array, it only contains objects, therefore me trying to pull specific variables wouldn't work.

What I realized is that I completely overcomplicated my code and instead of trying to rewrite the textfile based on specific variables, I can simply print out the object array back into the arraylist.

What Could be Improved

What could be improved is me further understanding how objects interact in arrays, object arrays, and so one. Furthermore, taking the time to understand and design functions before jumping in and code without thinking.
